
## PROJECT: LYOSINT — Libya OSINT SuperTool v3.0

Build a COMPLETE, FULL-STACK OSINT investigation platform on Replit targeting LIBYA exclusively. This tool searches for individuals by NAME, PHONE NUMBER, or USERNAME across Libyan-specific and international platforms with deep granularity, accuracy, and intelligence.

---

### 🎯 CORE MISSION
Create a unified, web-based intelligence dashboard that:
1. Takes a **Libyan name** → finds all associated phone numbers, usernames, addresses, family connections, social media profiles, and public records
2. Takes a **Libyan phone number (+218)** → identifies the owner, carrier (Libyana/Al-Madar/LibyaPhone), location district, and linked social media accounts
3. Takes a **username** → searches 400+ platforms (Sherlock-based) plus Libya-specific platforms (Facebook groups, Libyan forums, Telegram channels)

---

### 🏗️ ARCHITECTURE (Full Stack)

**BACKEND (Python FastAPI)**:
- FastAPI with async support
- Celery + Redis for task queue (background scraping)
- SQLite/PostgreSQL via SQLAlchemy for caching results
- WebSocket for real-time progress updates

**FRONTEND (React + Next.js or Vue.js)**:
- Tailwind CSS + DaisyUI for dark-themed hacker-style UI
- Real-time result streaming via WebSocket
- PDF/CSV export of investigation results
- Responsive design for mobile/desktop

### 📱 MODULE 2: PHONE NUMBER SEARCH (بحث برقم الهاتف)

**Input**: Libyan phone number `+218912345678` (also accepts local format `0912345678`)

**Libya-Specific Processing**:
1. **Format validation** using `phonenumbers` library with Libya (+218) rules
2. **Carrier identification**:
   - `091`, `093` → Al-Madar
   - `092`, `094` → Libyana
   - `095` → LibyaPhone
3. **Geographic inference** from prefix (some prefixes are region-specific)
4. **PhoneLibya.ly** — Scrape for owner name associated with number
5. **Truecaller lookup** (via unofficial API/scraper if available)
6. **Reverse phone search APIs** — NumLookup, FreeCarrierLookup, CallerID
7. **Google dork**: `"0912345678" site:.ly`, `"+218912345678" filetype:pdf`
8. **Social media search** — Search number across Telegram, Facebook, WhatsApp
9. **Facebook Graph search**: `phone:"0912345678"` in public Libyan groups
10. **Telegram phone search**: Using Telethon/MTProto to find user by phone
11. **Leaked DB search**: Check against known Libyan data breaches (if available)
12. **WhatsApp check**: Using whatsapp-web.js to verify if number is on WhatsApp

**Output**:

### 🔬 MODULE 4: DEEP CROSS-REFERENCE & CORRELATION ENGINE

This is the **JUICY PART** — the engine that ties everything together:

- **Phone ↔ Name ↔ Username Correlation**: Any search result that yields a phone number triggers automatic name search and username search
- **Family Tree Inference**: When the same last name + same address appears in multiple records, suggests family relationships
- **Network Mapping**: Graph visualization showing connections between people, phones, addresses, and usernames
- **Timeline Builder**: Chronological ordering of when each piece of data was first/last seen
- **Confidence Scoring**: Algorithm that weights results by source reliability, recency, and cross-references
- **Duplicate Merging**: Fuzzy string matching to merge similar names (Arabic/English variants)

---

### 🧠 MODULE 5: AI-ENHANCED DETECTION & ENRICHMENT

- **NLP for Arabic name parsing**: Handle "علي محمد المبروك" → identify first, middle, last, tribe/family name
- **Name variant generator**: Generate all possible spellings from Arabic → English (e.g., Mabrouk/Mabrook/Mabruq/Elmabrouk/Almabrouk)
- **OCR for scanned documents**: If PDFs/images are found in Google dork results, extract Arabic text
- **Sentiment/risk analysis**: Classify profiles as "high risk / low risk" based on content

### 🧪 IMPLEMENTATION STEPS ON REPLIT

1. **Create Replit Project**: Use "Python (FastAPI)" template or "Next.js" template
2. **Set environment variables** in `.env`:
   ```
   CELERY_BROKER_URL=redis://localhost:6379
   DATABASE_URL=sqlite:///lyosint.db
   SECRET_KEY=your-secret-key-here
   SHODAN_API_KEY=
   TELEGRAM_API_ID=
   TELEGRAM_API_HASH=
   INSTAGRAM_USERNAME=
   INSTAGRAM_PASSWORD=
   TWITTER_BEARER_TOKEN=
   ```
3. **Backend**: Create `backend/app/` structure and implement all routers
4. **Frontend**: Create React app with Tailwind, implement all views
5. **Sherlock integration**: Clone sherlock as submodule, wrap in Python API
6. **Test each module**: run `uvicorn backend.app.main:app --port 8080`
7. **Deploy**: Replit will auto-deploy on port 8080
